# 简易编译器
从更为理论，正规的方式来看，还是应该有一些前置的设计。这里我是笔记，所以编译器的第一步就是：
1. parser 解析器

这里，“代码”解释成抽象语法树AST。

## 对抗问题复杂化
关于如何保持复杂性可管理的一些建议：
* 编写多个函数，分解任务。在每项功能中，都要做一件事并做好。
* 不要在parser上尝试使用正则表达式
    * 正则表达式在 lexer 上可能效果不错
* 不要试图预测“源码”，错了，就throw error，别犹豫；

## parser 分解为三个问题
1. 字符串流输入，
2. token流（lexer）
3. parser
逐字分析，目标分析出token，token流逐步分析，生成ast，完成parser
